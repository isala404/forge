# Frontend Integration

FORGE generates TypeScript types from your Rust code and provides a Svelte 5 client with:

- **Type-safe API calls** - Full autocomplete for queries and mutations
- **Real-time subscriptions** - Data updates automatically via WebSocket
- **Job tracking** - Monitor background jobs in your UI
- **Optimistic updates** - Instant UI feedback

## Quick Example

```svelte
<script lang="ts">
  import { subscribe, mutate } from '$lib/forge';

  // Real-time subscription - updates when data changes!
  const tasks = subscribe('list_tasks', {});

  async function addTask(title: string) {
    await mutate('create_task', { title });
    // No need to refetch - subscription updates automatically
  }
</script>

{#if $tasks.loading}
  <p>Loading...</p>
{:else if $tasks.error}
  <p>Error: {$tasks.error.message}</p>
{:else}
  <ul>
    {#each $tasks.data ?? [] as task}
      <li>{task.title}</li>
    {/each}
  </ul>
{/if}
```

## What You Get

### Type Safety

Your Rust types become TypeScript types:

```rust
// Rust
#[derive(Serialize)]
pub struct Task {
    pub id: Uuid,
    pub title: String,
    pub completed: bool,
}
```

```typescript
// Generated TypeScript
interface Task {
  id: string;
  title: string;
  completed: boolean;
}
```

### Auto-complete

Full IDE support:

```typescript
// Query names are typed
const tasks = subscribe('list_tasks', {});
//                       ^^^^^^^^^ autocomplete!

// Input types are checked
await mutate('create_task', { title: 'Buy milk' });
//                          ^^^^^^^^ type-checked!
```

## Frontend Stack

FORGE works with **Svelte 5** and **SvelteKit**:

| Component | Version | Notes |
|-----------|---------|-------|
| Svelte | 5.x | Uses runes (`$state`, `$derived`) |
| SvelteKit | 2.x | File-based routing |
| TypeScript | 5.x | Full type generation |
