## Quick Example

Here's how simple FORGE is. Define your data:

```rust title="src/schema/mod.rs"
#[derive(Debug, Clone, Serialize, Deserialize, sqlx::FromRow)]
pub struct Task {
    pub id: Uuid,
    pub title: String,
    pub completed: bool,
    pub created_at: DateTime<Utc>,
}
```

Write a query:

```rust title="src/functions/queries/list_tasks.rs"
#[forge::query]
pub async fn list_tasks(ctx: &QueryContext) -> Result<Vec<Task>> {
    sqlx::query_as("SELECT * FROM tasks ORDER BY created_at DESC")
        .fetch_all(ctx.db())
        .await
        .map_err(Into::into)
}
```

Use it in your frontend:

```svelte title="src/routes/+page.svelte"
<script lang="ts">
  import { subscribe } from '$lib/forge';

  // This automatically updates when data changes!
  const tasks = subscribe('list_tasks', {});
</script>

{#each $tasks.data ?? [] as task}
  <div>{task.title}</div>
{/each}
```

**That's it.** The data automatically syncs when it changes. No WebSocket setup, no subscription management, no Redux.
